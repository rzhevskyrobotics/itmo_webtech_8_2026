## Задание

В рамках практической работы требовалось:

-   Развернуть экземпляр CMS WordPress.
-   Настроить NGINX для совместной работы WordPress и собственного
    Node.js-приложения.
-   Реализовать маршруты:
    -   `/login/`
    -   `/wordpress/`
    -   `/render/`
-   Обеспечить корректную работу REST API WordPress по субмаршруту\
    `/wordpress/wp-json/wp/v2/`.
-   Реализовать серверный рендеринг Pug-шаблона по URL, переданному в
    параметре `addr`.

## Краткое описание выполненной работы

В ходе выполнения задания реализовано:

-   Развернут VPS сервер Linux с установленным NGINX.
-   Настроена конфигурация реверсивной прокси для работы WordPress и
    Node.js-приложения.
-   Развернут экземпляр WordPress в подмаршруте `/wordpress/`.
-   Создан пост с идентификатором `1`, заголовок которого соответствует
    выданному логину.
-   Реализовано серверное приложение на Node.js (Express).
-   Настроен прием JSON-запросов и обработка query-параметров.
-   Реализован динамический рендер Pug-шаблона с подстановкой входных
    параметров.

## Архитектура решения

Схема взаимодействия компонентов:

Client\
→ NGINX\
  ├── `/login/` → Node.js (Express)\
  ├── `/render/` → Node.js (Express)\
  └── `/wordpress/` → WordPress (PHP-FPM)

NGINX выполняет функцию reverse proxy и маршрутизации запросов в
зависимости от URI (казалось бы, парадоксально, но это так).
